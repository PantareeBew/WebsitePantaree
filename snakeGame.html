Accesible Snake Game

“Inclusive Design doesn’t mean you’re designing one thing for all people. You’re designing a diversity of ways to participate so everyone has a sense of belonging.”

- Susan Goltsman

📋 Project Overview ✏️

  • Role: UX Designer, Researcher, and Accessibility Analyst
  • Methods: Usability testing, think-aloud protocol, personas
  • Deliverables: Accessibility evaluation report, personas, usability testing findings, design iteration recommendations, inclusive design strategy, high-fidelity prototypes
  • Skills: UX/UI design, user research, accessibility testing (WCAG compliance)
  • Tools: Microsoft Forms, Microsoft word, Figma, Microsoft Teams/Zoom, WebAIM contrast checker, Color Blindness Simulator

This project involved the end-to-end design and accessibility research of a Snake game application, ensuring inclusive experiences for users with visual, auditory, motor, and cognitive impairments. The game features four modes (Classic, Speed Run, Multiplayer, Custom) with multi-modal control options (mouse, keyboard, game controller), high-contrast UI elements, and customizable settings for accessibility.

I independently conducted research, UX/UI design, sketches, prototyping, and usability testing with five participants, identifying areas for improvement such as UI clarity in multiplayer mode, keyboard focus visibility, and distinguishability of game elements. Iterative design adjustments were made based on WCAG guidelines to ensure compliance in keyboard accessibility, contrast ratios, and audio settings, making the game more accessible for all users.

🧑‍🦽‍➡️Research & Personas

To ensure the game was accessible, I created personas representing users with different impairments. Each persona helped me identify specific challenges and solutions to implement in the game design.

Peersona Impairment Key Challenges Design Needs
🧑‍💻 Oiver Protanopia (Color Blindness) Can't differentiate red/green UI elements High contrast UI, pattern-based differentiation
🧔🎮 Alex Motor Impairment (Parkinson’s) Struggles with small buttons & precise movements Large buttons, keyboard navigation
🎧🦻 Emily Hearing Impairment Can't rely on sound cues Visual notifications, subtitles for audio feedback
🧠👧 Maya Cognitive Impairment (ASD) Overwhelmed by cluttered UI Predictable navigation, structured layout


🔬👩‍🎨Research & Design Process

My research phase included:

  • 🔍 Reviewing accessibility principles, including Medical vs. Social Models of Disability
  • 📑 Studying assistive technology and its impact on digital experiences
  • 🎯 Conducting a comparative analysis of accessible and non-accessible games

After the research phase, I moved into UX/UI design:

  • 1️⃣ Sketches & Wireframes – Created low-fidelity wireframes to structure the interface
  • 2️⃣ High-Fidelity Prototypes –Designed the final UI, ensuring accessibility best practices
  • 3️⃣ Game UI Considerations –Designed with simple navigation, scalable text, high-contrast colors, and customizable controls

📉Usability Testing Results & Insights

🎮 Unclear Button Label

  • Issue Identified
    The button "random again" was not clear between randomising some people to join the game or randomising the skin colour.
  • Design Solution
    Clearer button labeling and tooltips to specify function.

🕹️ Grid-based UI in Multiplayer Overwhelms Users

  • Issue Identified
    Players felt the interface was cluttered, making it difficult to focus.
  • Design Solution
    Simplified UI with spacing adjustments and visual hierarchy improvements.

🐍 Snakes and Obstacles Look Too Similar

  • Issue Identified
    Players struggled to differentiate between their snake and obstacles.
  • Design Solution
    Introduced distinct shapes and outlines for obstacles.

🔍 Difficulty Identifying Own Snake in Multiplayer

  • Issue Identified
    Users couldn't easily recognize which snake belonged to them.
  • Design Solution
    Added player name tags and subtle glow effects.

⌨️ Keyboard Focus Visibility Issues on Skin Selection Screen

  • Issue Identified
    Focus indicators were not prominent, leading to confusion.
  • Design Solution
    Improved focus visibility with bold highlights and contrasting outlines.

🌓Key Takeaways & Reflections

After multiple design iterations, the final game UI meets accessibility standards by implementing:

  • 🌟What I Learned: Accessibility-first design makes games more enjoyable for all users
  • 🌟Biggest Challenge: Balancing game mechanics, accessibility features, and intuitive design
  • 🌟Next Steps: Applying inclusive design principles to future projects and refining accessibility strategies
